# SD Forge Image Stitch - 常用编辑提示词与批量处理功能

## 📝 功能概述

在 `sd_forge_image_stitch` 插件中新增了**常用编辑提示词管理**和**批量图片处理**功能。批量处理功能可以对指定目录下的每一张图片分别进行编辑处理，生成对应的输出图片。

## ✨ 主要特性

### 1. **预设提示词收藏**
- 用户可以添加、删除和管理常用的编辑提示词
- 所有提示词持久化保存在本地，重启后依然可用

### 2. **默认提示词**
首次运行时会自动创建以下7个常用提示词：
- 改为3d灰模
- 生成三视图，正面，侧面，背面
- 只参考姿势
- 去除背景
- 去除文字
- 4k画质
- 二次元动漫风格

### 3. **简洁列表展示**
- 使用 Dropdown 下拉列表展示所有预设提示词
- 纯文本显示，清晰直观，无图片卡片
- 支持中文显示

### 4. **便捷操作**
- ➕ **添加**：输入新提示词并保存
- 🗑️ **删除**：从下拉列表选择后删除该提示词
- 🔄 **刷新**：重新加载提示词列表

### 5. **⭐ 智能传递到主程序**
- 自动检测当前是文生图还是图生图模式
- 一键将提示词复制到对应的正面提示词输入框中
- 复制成功后右上角弹出绿色通知提示
- 通知3秒后自动消失，不干扰操作

### 6. **🔄 批量编辑生成（新功能）**

#### ⚠️ 重要提示
**此功能仅适用于支持图像编辑的模型**，例如：
- **Flux-Kontext**
- **Flux.2-Klein**
- **Qwen-Image-Edit**

这些模型能够根据提示词对输入图片进行智能编辑和重绘。使用其他模型可能无法获得预期效果。

#### 基本流程
1. 展开 **"🔄 批量图片处理"** 折叠面板
2. 配置处理参数：
   - **输入目录**：包含待处理图片的文件夹路径（如：`D:/images/input`）
   - **输出目录**：处理后图片的保存位置（如：`D:/images/output`）
   - **编辑提示词**：应用于所有图片的编辑指令
   - **负面提示词**（可选）：需要避免的内容描述
3. 调整生成参数：
   - **采样步数**：推荐 20-40 步
   - **CFG Scale**：推荐 7.0-9.0
   - **种子**：-1 表示随机，或指定固定种子保证可复现
4. 设置文件过滤：
   - **支持的文件格式**：勾选需要处理的图片格式
   - **处理顺序**：选择图片的处理顺序
5. 点击 **"🚀 开始批量处理"** 按钮
6. 等待处理完成，查看日志和进度
7. 在输出目录中查看处理后的图片

#### 💡 批量处理使用场景
- **模型训练数据增强**：对一批原始图片应用统一的编辑效果
- **风格转换批次处理**：将多张图片转换为特定风格（如3D灰模、动漫风格等）
- **姿态标准化**：批量调整图片的姿态或视角
- **质量优化**：对低质量图片进行批量修复和增强
- **格式转换**：结合提示词实现智能化的图片重绘和格式统一

#### ⚙️ 参数说明
- **目标最大边长**：输出图片的目标最大边长（0表示保持原图尺寸）
  - **小图放大**：当原图小于设定值时，会自动放大到目标尺寸（如 512x512 → 1024x1024）
  - **大图缩小**：当原图大于设定值时，会自动缩小到目标尺寸（如 2048x1536 → 1024x768）
  - **保持比例**：自动保持原图宽高比，并调整为 **16 的倍数**（FLUX VAE 编码器要求）
  - **推荐设置**：512-1024（平衡质量和显存占用）
- **去噪强度**：默认为 0.75，控制原图保留程度（值越大变化越大）
- **采样步数**：影响生成质量和速度，步数越多质量越好但耗时越长
- **CFG Scale**：控制提示词遵循程度，值越高越严格遵循提示词
- **种子**：固定种子可以保证相同输入产生相同输出，便于调试

#### 📊 日志解读
处理日志会显示：
- 当前处理进度（X/Y 张）
- 每张图片的处理状态（✅ 成功 / ❌ 失败）
- 错误详情（如果处理失败）
- 最终统计（总计/成功/失败数量）

### 清空批量列表
1. 展开 **"📁 批量编辑"** 折叠面板
2. 点击 **"🗑️ 清空列表"** 按钮
3. 所有参考图片将被清空

### 删除提示词：
1. 从下拉列表选择要删除的提示词
2. 点击 **"🗑️ 删除选中提示词"** 按钮
3. 提示词会被永久删除

### 刷新列表
- 点击 **"🔄 刷新列表"** 按钮重新加载所有预设提示词
- 适用于手动修改了 presets 目录文件后

## 📂 数据存储

所有预设提示词保存在以下目录：
```
sd-webui-forge-neo-v3/webui/extensions-builtin/sd_forge_image_stitch/presets/
```

每个提示词以 `.txt` 文件形式存储：
- 文件名：提示词的前20个字符（特殊字符会被替换为 `_`）
- 文件内容：完整的提示词文本
- 编码格式：UTF-8

**示例文件结构：**
```
presets/
├── 改为3d灰模.txt
├── 生成三视图，正面，侧面，背面.txt
├── 只参考姿势.txt
├── 去除背景.txt
├── 去除文字.txt
├── 4k画质.txt
└── 二次元动漫风格.txt
```

## 💡 使用场景

### 1. **3D 建模辅助**
```
改为3d灰模，生成三视图，正面，侧面，背面
```
用于快速生成角色的多角度参考图

### 2. **图像清理**
```
去除背景，去除文字
```
清理素材中的干扰元素

### 3. **风格转换**
```
二次元动漫风格，4k画质
```
统一输出图像的风格和质量

### 4. **姿势参考**
```
只参考姿势
```
提取人物动作而不受其他因素影响

## 🔧 技术实现

- **UI 框架**：Gradio Dropdown + Textbox + Button
- **文件管理**：基于文件系统的持久化存储（`presets/` 目录下的 .txt 文件）
- **事件绑定**：Gradio 响应式事件处理

## ⚠️ 注意事项

1. **提示词长度**：建议单个提示词不超过200个字符，以保持下拉列表的可读性
2. **特殊字符**：文件名会自动处理 `/ \ :` 等特殊字符
3. **重复添加**：如果提示词已存在，系统会自动添加序号后缀（如 `改为3d灰模_1.txt`）
4. **浏览器兼容性**：复制到主界面功能依赖现代浏览器（Chrome、Edge、Firefox）
